Build a production-ready full-stack SaaS application called **PulseFlow**.

PulseFlow is an **AI WhatsApp Front Desk for solo beauty & wellness businesses in India** (salons, home beauticians, makeup artists, nail artists, skincare/wellness studios). It is **not** a generic CRM. It is a vertical workflow automation product that turns WhatsApp conversations into bookings, reminders, review requests, and repeat-customer follow-ups.

I want you to actually implement the app in code inside this Replit project. Do not give me only a product plan or architecture notes. Build the working codebase step by step.

---

# 1) Product Goal

PulseFlow should help a solo beauty business owner:

1. set up their business profile, services, working hours, pricing, policies, and review link
2. receive customer WhatsApp inquiries
3. let AI answer repetitive questions about services, pricing, availability, and location
4. capture booking intent from customer chats
5. convert booking requests into structured appointments
6. send booking confirmations
7. send 24h and 2h reminders automatically
8. send post-service review requests automatically
9. send repeat-visit nudges after a configurable interval
10. allow owner review/approval for uncertain AI actions
11. manage everything from a clean mobile-first dashboard

# 12) Automation Engine (Very Important)

Build a clean, modular, idempotent automation engine.

Create folder:
- /lib/automations/

Required files:
- automation-service.ts
- automation-types.ts
- automation-registry.ts
- template-service.ts or use /lib/templates/message-templates.ts

Implement these core functions:

## createAutomationEvent()
Creates a scheduled automation event in automation_events.

Requirements:
- generate a deterministic idempotencyKey
- if same event already exists, do not create duplicate
- return existing event if duplicate found

## scheduleBookingAutomations(booking)
Triggered when booking is confirmed.
Should create:
- booking_confirmation (immediate)
- reminder_24h
- reminder_2h

Must be idempotent and avoid duplicates.

## scheduleCompletionAutomations(booking)
Triggered when booking is completed.
Should create:
- review_request
- repeat_visit_nudge

Use service.repeatReminderDays for repeat reminder scheduling.

## processDueAutomationEvents(now = new Date())
This is the job runner heart.
It should:
1. fetch all pending automation events with scheduledFor <= now
2. process each safely
3. mark status processing -> sent/failed/skipped
4. increment attemptCount
5. save lastAttemptAt
6. never send duplicates
7. return summary:
   { processed, sent, failed, skipped }

## Automation event types:
- booking_confirmation
- reminder_24h
- reminder_2h
- review_request
- repeat_visit_nudge

## Message templates
Support variables:
- {name}
- {service}
- {date}
- {time}
- {business}
- {review_link}

Create default templates for:
- booking confirmation
- 24h reminder
- 2h reminder
- review request
- repeat visit nudge

Use a clean template rendering helper.
