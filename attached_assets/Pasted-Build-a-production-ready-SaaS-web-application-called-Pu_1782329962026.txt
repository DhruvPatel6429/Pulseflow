Build a production-ready SaaS web application called **PulseFlow**.

## Product Overview
PulseFlow is an **AI WhatsApp front desk for solo beauty & wellness businesses in India**. It helps salons, beauticians, makeup artists, nail artists, and small wellness studios automate customer communication and operational workflows on WhatsApp.

PulseFlow should not be built as a generic CRM. It should be built as a **vertical AI workflow product** focused on turning WhatsApp inquiries into:
- confirmed bookings
- automated reminders
- reduced no-shows
- post-service review requests
- repeat-visit follow-up nudges

The target users are solo beauty/wellness businesses with 1–5 staff who currently manage bookings manually on WhatsApp.

---

# Primary Goal
Build a **usable production-grade MVP** of PulseFlow that a real pilot customer could use.

The system must allow a beauty business owner to:
1. configure their business profile, services, prices, durations, working hours, and booking rules
2. connect a WhatsApp number via WhatsApp Business Cloud API integration-ready architecture
3. let customers message on WhatsApp and receive AI-generated responses
4. capture booking requests and convert them into structured appointments
5. send booking confirmations and reminders
6. trigger post-service review requests
7. trigger repeat-visit reminders after configurable intervals
8. give the owner a lightweight dashboard to monitor and override AI actions when needed

---

# Non-Negotiable Product Positioning
This is NOT a horizontal CRM.
This is an **AI WhatsApp Front Desk for solo beauty & wellness businesses**.

The UX, database, terminology, and workflows must reflect this niche:
- beauty services
- appointment durations
- slot booking
- reschedule / cancellation
- bridal / premium services with optional token payments later
- review requests
- repeat visit reminders

# Build Requirements
I want you to build this as a **production-ready full-stack app**, not a hackathon prototype.

## Use the following stack
- **Frontend:** Next.js 14+ with TypeScript and App Router
- **UI:** Tailwind CSS + shadcn/ui
- **Backend/API:** Next.js server actions / route handlers OR a clean Node backend inside the same repo if needed
- **Database:** PostgreSQL with Prisma ORM
- **Auth:** NextAuth or Clerk (choose the simplest production-grade option)
- **Background jobs / scheduling:** use a reliable queue/scheduler architecture suitable for reminders and delayed workflows. If Replit has limitations, create a clean abstraction layer that can later be swapped to Upstash/QStash, BullMQ, Trigger.dev, or a cron worker.
- **AI:** OpenAI API abstraction layer with clean prompt orchestration
- **WhatsApp integration:** architect around Meta WhatsApp Cloud API webhook + send-message flows, even if sandbox mode/mock mode is needed first
- **Validation:** Zod
- **State / forms:** React Hook Form where appropriate
- **Logging:** structured logging
- **Environment config:** proper `.env.example`
- **Error handling:** production-safe error boundaries and API error responses
- **Deployment readiness:** codebase should be clean, typed, modular, and easy to deploy

# 5. WhatsApp Integration Architecture
Build the codebase so it is ready for Meta WhatsApp Cloud API.

## Implement:
- a webhook endpoint for inbound WhatsApp messages
- a service layer that normalizes inbound messages into internal Conversation + Message records
- an outbound messaging service abstraction
- provider adapters architecture, even if only WhatsApp is used now

### Important:
If direct live WhatsApp API setup is difficult inside Replit, implement:
1. a real integration-ready service layer
2. a **mock/sandbox mode** for testing from the dashboard
3. clear comments and environment variables for how to connect the real WhatsApp Cloud API later

Create files/modules like:
- `/lib/channels/whatsapp.ts`
- `/lib/channels/types.ts`
- `/app/api/webhooks/whatsapp/route.ts`

The architecture must not be tightly coupled to one transport implementation.

# 6. AI Inquiry Responder
This is the heart of the product.

Build an AI orchestration layer that reads inbound customer WhatsApp messages and classifies them into business intents such as:
- service price inquiry
- availability inquiry
- booking request
- reschedule request
- cancellation request
- location inquiry
- package inquiry
- follow-up / post-service question
- unknown / escalate

## The AI layer must:
1. use the business knowledge base (services, prices, durations, policies, working hours, location, FAQs)
2. understand customer messages in natural language
3. produce a structured response object, not just free text

### Example AI output structure
```ts
type AIIntentResult = {
  intent:
    | "price_inquiry"
    | "availability_inquiry"
    | "booking_request"
    | "reschedule_request"
    | "cancel_request"
    | "location_inquiry"
    | "faq"
    | "unknown";

  confidence: number;

  extractedEntities: {
    serviceName?: string;
    requestedDate?: string;
    requestedTime?: string;
    customerName?: string;
  };

  actionSuggestion: {
    shouldReply: boolean;
    shouldCreatePendingBooking: boolean;
    shouldAskFollowUpQuestion: boolean;
    shouldEscalateToOwner: boolean;
  };

  replyDraft: string;
};
